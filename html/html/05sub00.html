주요 내용

git / github

CMD 내부에서의 기본적인 명령어

git git 명령어 설명서 출력
cd d: d 드라이브로 이동
pwd 현재 위치 출력
mkdir 폴더명 현재 위치에 폴더만들기
ls -al 현재 폴더의 파일목록
clear 창 내용 클리어
git init 현재 폴더를 git의 저장소로 만듬 (초기화)
vim 파일명.확장자명 파일명 새것 = 파일 생성 / 파일명 중복 = 수정
  • i : insert 모드 변환
  • esc : insert 모드 종료
  • r : 자세한건 모르겠는데 포커스 된 곳만 수정가능한듯?
  • : (콜론) : 명령어 모드 변환
    • w : 저장
    • q : 종료
    • wq : 저장 후 종료
cat 파일명.확장자명 내용 출력
git status 현재 git이 관리하고 있는 파일에 대한 상태
git add 파일명.확장자명 최초 버전관리 시작 = git에게 '추적' 하라는 명령 내림 / 수정한 파일을 'Staged' 상태로 만듬
git config --global user .name 이름 버전 만든이 명시
git config --global user .email 이메일주소 버전 만든이 이메일주소 명시 맨처음 한번만 하면 나머지 버전에 적용 + 다른 사용자가 해당 버전의 만든이를 알 수 있음.
또한 수정하려면 동일한 명령어를 적으면 덮어쓰는 개념임.
git commit 현재 버전에 대한 메시지 작성 # 은 주석임
어떠한 변화 / 변경된 이유
vim 실행상태가 됨
git log 버전관리 기록 출력
cp 파일명1.확장자명 파일명2.확장자명 파일명1을 복사하여 파일명2 라는 새로운 파일을 생성
git log -p 버전관리 기록 자세히
git reset 고유주소 --hard 고유주소를 최신버전으로 함 (상위 버전은 어딘가에 남아있음)
git revert 고유주소 나중에
q git log 읽기모드? 같은 모드에서 나가기
git commit --help commit 메시지에 대한 도움말 보기
git commit -a 수정/삭제된 파일을 자동으로 'Staged' 상태로 만듬
git commit -m "커밋메시지" commit 메시지 작성모드 대신에 간략하게 작성 하는것
git commit -am "커밋메시지" 둘이 합친거
git clone 복제할주소 폴더명 주소에 있는 원격저장소를 '폴더명' 에 복제 (git init 과 동일선상임) 현재폴더에다 할거면 '.' 만 적으3
git log --reverse 거꾸로 버전관리 기록 출력
git checkout 커밋고유주소 일단 패스
echo 옵션 문자열 환경변수 / 입력내용 출력 = 지정한 문자열을 출력함
git remote add origin 주소 로컬저장소에 해당 주소(=원격저장소)를 연결시키는 것 origin = 주소에 대한 별칭임
git remote 원격저장소 정보보기
git remote -v 원격저장소 정보 상세보기
git remote --help remote 명령어에 대한 도움말보기
git remote remove 별칭 해당 원격저장소 삭제
git push -u origin master 로컬저장소의 작업을 원격저장소로 올림 로컬저장소의 브랜치와 origin 원격저장소의 마스터 브랜치를 서로 연결 시켜서, 다음번에 사용할때는 git push만 입력할수있게 하는 명령어
